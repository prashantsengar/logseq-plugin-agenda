# 一般自定义日历

TIP

一般自定义日历依据 query 收集事件并显示到日历上

当我们新建一个自定义日历后, 行为同 journal 默认日历, 但是查找范围变为了自定义日历 id 指定的页面

所有日历的 query 都是开放可修改的, 你可以根据需求自己定制

那么如何定制玩去属于自己的日历呢?

答案是新建日历,然后修改 queryopen in new window

插件会以填写的 query script 作为参数调用logseq.DB.datascriptQueryopen in new window 或 logseq.DB.qopen in new windowAPI, 然后将结果展示在日历中.

让我来解释一下有哪些配置项:

  1. script: 作为 datascriptQuery 的参数, 查询所有符合要求的 block
  2. query type: query 的类型, 可以是 Simple Queryopen in new window 或 Advanced Queryopen in new window
  3. schedule start: 从 datascriptQuery 查询的 block 取出 schedule start 指定的字段作为事件开始时间
  4. schedule end: 从 datascriptQuery 查询的 block 取出 schedule end 指定的字段作为事件结束时间
  5. date formatter: 日期格式, 以此为参数使用 date-fnsopen in new window 将 schedule start schedule end 转换为可用的日期
  6. is milestone: 是否是里程碑, 如果是, 则会展示在日历的 Milestone 中

如果你不知道如何填写 schedule start schedule end, 可以打开控制台, 然后点击 play 按钮:

# 示例:

当前我们有一个 test-agenda 的笔记:

其中 custom calendar demo 具有 start end 属性, 我们想让它显示在日历中,而 common text 不显示.

# 方法一: 使用 Simple Queryopen in new window

我们使用如下 query script 查询位于 test-agenda 页面中的 block:

(and (page "test-agenda") (property end) (property start))

# 方法二: 使用 advanced queryopen in new window

如果你更喜欢advanced query, 配置如下:

[:find (pull ?block [*])
  :where
  [?block :block/properties ?p]
  [(get ?p :start) ?s]
  [(get ?p :end) ?e]
  [?page :block/name ?pname]
  [?block :block/page ?page]
  [(contains? #{"test-agenda"} ?pname)]]
